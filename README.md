# 🚀 网页自动刷新工具

一个本地运行的网页访问与刷新测试工具。项目使用 Flask 提供 Web 操作界面，通过 Selenium 调用本机 Chrome 浏览器执行访问任务，适合用于自有网站的页面访问、刷新、统计埋点和基础功能测试。

> 所有任务都在本机执行，不依赖外部服务，也不会上传你的任务配置。

## ✨ 功能亮点

- 🖥️ **图形化操作界面**：打开网页即可配置和启动任务
- 🌐 **多任务支持**：可同时配置多个网址，每个任务参数独立
- 🔁 **PV / UV 两种模式**：支持单窗口重复刷新，也支持多独立窗口访问
- 👀 **可见 / 后台运行**：可弹出 Chrome 观察过程，也可无头静默运行
- ⚡ **并行执行**：支持设置并行数，提高批量测试效率
- 📊 **实时进度**：展示总进度、单任务进度、成功次数和失败次数
- 🛑 **随时停止**：运行中可停止任务并关闭当前浏览器实例
- 🎭 **随机 UA 画像**：UV 模式会随机选择浏览器 User-Agent 和窗口尺寸

## 🧭 访问模式

### 🔁 单窗口刷 PV

启动一个 Chrome 浏览器窗口，先打开目标网页，然后在同一个窗口内重复刷新。

适合测试：

- 页面刷新稳定性
- PV 统计
- 页面埋点触发
- 刷新后的页面状态

### 🧩 多窗口刷 UV

每次访问都会启动一个新的独立 Chrome 会话，使用独立临时用户目录和无痕模式，访问完成后自动关闭。

UV 模式会：

- 为每次访问创建独立浏览器会话
- 随机选择浏览器 User-Agent 画像
- 使用匹配的窗口尺寸
- 等待页面基础渲染完成
- 停留一小段时间后再关闭窗口

UV 停留时间可在 `app.py` 中调整：

```python
UV_MIN_STAY_SECONDS = 1.5
UV_MAX_STAY_SECONDS = 2.5
```

如果目标网站统计延迟较高，可以适当调大这两个值。

## 🖥️ 运行模式

### 👀 可见窗口

正常弹出 Chrome 浏览器窗口，可以观察每次访问和刷新过程，适合调试和确认页面表现。

### 🌙 后台静默

使用 Chrome 无头模式运行，不弹出浏览器窗口，适合批量测试或不想影响桌面操作时使用。

## 📋 环境要求

- Windows 或 macOS
- Python 3.8+
- 已安装 Google Chrome
- 可正常联网安装 Python 依赖

## 📦 安装依赖

在项目目录执行：

```bash
pip install -r requirements.txt
```

依赖包括：

- `Flask`
- `Selenium`

Selenium 4.6+ 会自动管理 ChromeDriver，通常不需要手动下载驱动。

## 🚀 启动方式

### 🍎 macOS

双击运行：

```text
start.command
```

脚本会启动本地服务，并自动打开：

```text
http://127.0.0.1:5000
```

### 🪟 Windows

双击运行：

```text
start.bat
```

脚本会启动本地服务，并自动打开：

```text
http://127.0.0.1:5000
```

### 🧑‍💻 手动启动

也可以在项目目录执行：

```bash
python app.py
```

然后在浏览器打开：

```text
http://127.0.0.1:5000
```

## 📝 使用步骤

1. 打开操作页面 `http://127.0.0.1:5000`
2. 输入目标网址
3. 选择功能模式：单窗口刷 PV 或多窗口刷 UV
4. 选择运行模式：可见窗口或后台静默
5. 设置访问次数
6. 设置并行数，建议先从 1-3 开始测试
7. 点击「开始执行所有任务」
8. 观察总进度和每个任务的成功、失败次数
9. 如需中断任务，点击「停止所有任务」

## 🧱 多任务使用

点击「新增任务」可以添加多个目标网址。每个任务都可以独立设置：

| 配置项 | 说明 |
| --- | --- |
| 目标网址 | 要访问或刷新的页面地址 |
| 功能模式 | 单窗口刷 PV / 多窗口刷 UV |
| 运行模式 | 可见窗口 / 后台静默 |
| 访问次数 | 当前任务要执行的总次数 |
| 并行数 | 同时运行的浏览器数量 |

所有任务会并行执行。并行数越高，占用的 CPU、内存和浏览器进程越多，建议根据电脑性能逐步调高。

## 🛑 停止服务

### macOS

关闭 `start.command` 打开的终端窗口，或在终端按：

```text
Ctrl + C
```

### Windows

关闭 `start.bat` 打开的命令行窗口即可。

## ⚙️ 常用配置

UV 模式页面 ready 后的额外停留时间：

```python
UV_MIN_STAY_SECONDS = 1.5
UV_MAX_STAY_SECONDS = 2.5
```

页面加载超时时间：

```python
PAGE_LOAD_TIMEOUT_SECONDS = 30
PAGE_READY_TIMEOUT_SECONDS = 15
```

这些配置都在 `app.py` 顶部附近。

## ⚠️ 注意事项

- 本工具仅用于自有网站或已授权场景下的测试
- 高并行会大量启动 Chrome 进程，可能占用较多系统资源
- UV 是否被目标网站统计，取决于目标网站的统计规则、埋点触发时机、IP、Cookie、浏览器指纹等因素
- 后台无头模式和可见窗口模式的统计结果可能存在差异，建议根据目标网站实际表现选择运行方式
